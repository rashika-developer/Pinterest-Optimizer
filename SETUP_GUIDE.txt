2. PROJECT SETUP - STEP BY STEP
================================================================================

STEP 1: Create Android Project
-------------------------------
1. Open Android Studio
2. Click "New Project"
3. Select "Empty Activity"
4. Configure Project:
   - Name: PinterestOptimizer
   - Package: com.pinterest.optimizer
   - Save Location: Choose your folder
   - Language: Kotlin
   - API Level: 24 (minimum)

STEP 2: Copy All Code Files
----------------------------
After project creation:
1. Download all .kt files from this guide
2. Copy them to: app/src/main/java/com/pinterest/optimizer/

File Organization:
- MainActivity.kt → app/src/main/java/com/pinterest/optimizer/
- models.kt → app/src/main/java/com/pinterest/optimizer/data/models/
- database.kt → app/src/main/java/com/pinterest/optimizer/data/database/
- dao.kt → app/src/main/java/com/pinterest/optimizer/data/database/
- api.kt → app/src/main/java/com/pinterest/optimizer/data/api/
- repository.kt → app/src/main/java/com/pinterest/optimizer/data/repository/
- viewmodel.kt → app/src/main/java/com/pinterest/optimizer/ui/viewmodel/
- di_module.kt → app/src/main/java/com/pinterest/optimizer/di/
- HomeScreen.kt → app/src/main/java/com/pinterest/optimizer/ui/screens/
- NicheProfileScreen.kt → app/src/main/java/com/pinterest/optimizer/ui/screens/
- PostCreationScreen.kt → app/src/main/java/com/pinterest/optimizer/ui/screens/
- AnalyticsScreen.kt → app/src/main/java/com/pinterest/optimizer/ui/screens/
- Theme.kt → app/src/main/java/com/pinterest/optimizer/ui/theme/

STEP 3: Update build.gradle Files
-----------------------------------
1. Replace app/build.gradle with the provided build.gradle file
2. Sync Gradle when prompted (File → Sync Now)
3. Wait for dependencies to download (may take 5-10 minutes)

STEP 4: Update AndroidManifest.xml
-----------------------------------
1. Replace app/src/main/AndroidManifest.xml with provided version
2. Ensure all permissions are included

STEP 5: Create res Resources
-----------------------------
Create these resource files:

app/src/main/res/values/strings.xml:
```xml
<?xml version="1.0" encoding="utf-8"?>
<resources>
    <string name="app_name">Pinterest Optimizer</string>
    <string name="permission_camera">Camera</string>
    <string name="permission_storage">Storage Access</string>
</resources>
```

app/src/main/res/values/themes.xml:
```xml
<?xml version="1.0" encoding="utf-8"?>
<resources>
    <style name="Theme.PinterestOptimizer" parent="android:Theme.Material.Light">
    </style>
</resources>
```

Create needed directories:
- app/src/main/res/drawable/
- app/src/main/res/mipmap/
- app/src/main/res/xml/

STEP 6: Add Application Class
------------------------------
Create file: app/src/main/java/com/pinterest/optimizer/PinterestOptimizerApp.kt

```kotlin
package com.pinterest.optimizer

import android.app.Application
import dagger.hilt.android.HiltAndroidApp

@HiltAndroidApp
class PinterestOptimizerApp : Application()
```

Update AndroidManifest.xml to include:
```xml
<application
    android:name=".PinterestOptimizerApp"
    ...
>
```

3. RUNNING THE APP
================================================================================

METHOD 1: Using Android Emulator
---------------------------------
1. Android Studio → Device Manager (left sidebar)
2. Click "Create Device"
3. Select Pixel 4a (or any modern device)
4. Choose Android 12 or later API level
5. Click "Create" and wait for installation
6. Click the Play button to start emulator
7. In Android Studio: Run → Run 'app' (or press Shift + F10)

METHOD 2: Using Physical Device
--------------------------------
1. Enable Developer Mode on Android phone:
   - Settings → About Phone
   - Tap "Build Number" 7 times
   - Developer options will appear

2. Enable USB Debugging:
   - Settings → Developer Options → USB Debugging

3. Connect phone to PC with USB cable
4. In Android Studio: Run → Run 'app'
5. Select your device from the dialog
6. App will install and launch

App should load showing the Home screen with:
- Welcome message
- Quick action buttons
- Feature descriptions

4. BACKEND CONFIGURATION
================================================================================

BACKEND SETUP OPTIONS:

A) EASIEST - Firebase (Serverless)
-----------------------------------
1. Go to https://console.firebase.google.com
2. Create new project
3. Enable these services:
   - Cloud Functions
   - Realtime Database (or Firestore)
4. Deploy Cloud Functions using the provided backend code
5. Update API base URL in di_module.kt

B) MEDIUM - Node.js + Express
------------------------------
1. Create backend folder:
   mkdir pinterest-optimizer-backend
   cd pinterest-optimizer-backend

2. Initialize Node project:
   npm init -y

3. Install dependencies:
   npm install express cors dotenv axios body-parser

4. Create server.js with provided code

5. Create .env file:
   PORT=5000
   OPENAI_API_KEY=sk-xxxxx (from https://platform.openai.com)
   NODE_ENV=development

6. Start server:
   node server.js

7. Test endpoints using Postman:
   - POST http://localhost:5000/api/generate-captions

8. Deploy to Heroku:
   heroku login
   heroku create your-app-name
   git push heroku main

UPDATE ANDROID APP:
-------------------
In di_module.kt, change:
```kotlin
.baseUrl("https://your-app-name.herokuapp.com/")
```

Or for local testing:
```kotlin
.baseUrl("http://10.0.2.2:5000/") // 10.0.2.2 is localhost from emulator
```

C) ADVANCED - AWS Lambda
------------------------
1. Create AWS account
2. Create Lambda functions for each endpoint
3. Create API Gateway to expose functions
4. Use provided code as Lambda handler

REQUIRED API KEYS:
-------------------
1. OpenAI API (for caption/hashtag generation):
   - Sign up: https://platform.openai.com
   - Create API key
   - Cost: ~$0.002 per caption (very cheap)

2. Amazon Associates (for product links):
   - Sign up: https://affiliate-program.amazon.com/
   - Get Associate ID
   - Earn commissions on referred products

3. Pinterest API (optional, for Pinterest integration):
   - Register: https://developers.pinterest.com/
   - Required only for direct Pinterest posting

5. FEATURES EXPLAINED
================================================================================

FEATURE 1: Niche Profile Setup
-------------------------------
User Action: Home → "Niche Profile" → Select category → Add description
Flow:
1. User selects from 10 predefined niches (Home Decor, Fashion, etc.)
2. App stores selection in local Room database
3. All future generations filtered by this niche
4. Example: Kitchen image → Home Decor user gets "modern kitchen" captions
           vs Fashion user gets "kitchen as fashion statement"

Implementation:
- NicheProfileScreen shows list of niche options
- NicheProfileViewModel saves to database via NicheProfileRepository
- Repository uses NicheProfileDao for database operations

FEATURE 2: Caption Generation (3 Variations)
----------------------------------------------
User Action: Create Post → Describe image → "Generate Captions"
Flow:
1. User enters image description (can also upload image)
2. App sends description + niche to backend API
3. Backend uses OpenAI to generate 3 different caption styles:
   - Minimal & Aesthetic: "Clean. Simple. Timeless."
   - Engaging & Conversational: "Just finished this kitchen makeover! What do you think?"
   - Sales-Oriented: "Get this look! Link in bio 👇"
4. User selects their favorite caption

Cost: ~$0.002 per generation (using OpenAI API)

Implementation:
- PostCreationScreen collects user input
- PostCreationViewModel calls ContentGenerationRepository
- Repository makes API call via PinterestOptimizerApi (Retrofit)
- Backend generates captions using OpenAI's GPT API
- Results displayed in CaptionOptionCard components

FEATURE 3: Hashtag Generation
------------------------------
User Action: Create Post → "Generate Hashtags"
Flow:
1. App analyzes image/description
2. Generates 15 relevant hashtags based on:
   - User's niche
   - Current Pinterest trends
   - Content category
3. Shows "regular" + "trending" hashtags separately
4. User can copy or edit hashtags

Implementation:
- Similar to caption generation
- Uses HashtagRequest data class
- Backend queries trending hashtag database
- Results shown in HashtagDisplay component

FEATURE 4: Product Link Finder
-------------------------------
User Action: Create Post → "Find Product Links"
Flow:
1. App analyzes image/description
2. Searches affiliate networks for matching products:
   - Amazon Associates
   - Etsy
   - Other affiliate networks
3. Returns products with:
   - Title & image
   - Affiliate link
   - Source (Amazon, Etsy, etc.)
4. User selects products to add to post

Revenue: Earn commission when users click product links
Example: Home decor post → Find matching furniture → User clicks link → You earn 4-10% commission

Implementation:
- ProductRequest sends image description to backend
- Backend queries affiliate API databases
- Results mapped to ProductLink model
- Displayed in ProductCard components

FEATURE 5: Analytics Tracking
------------------------------
User Action: Home → "Analytics"
Flow:
1. Tracks performance metrics:
   - Views (impressions)
   - Saves (bookmarks)
   - Clicks (to product links)
   - Engagement rate
2. Aggregates data by post
3. Shows trends over time
4. Helps user optimize future posts

Data Collection:
- Integrated with Pinterest API
- Or manually logged when users view stats in Pinterest

Implementation:
- AnalyticsViewModel fetches from AnalyticsRepository
- Data stored in PostAnalytics Room entity
- Displayed via AnalyticsScreen with StatCard components

FEATURE 6: Board Auto-Suggestion
-------------------------------
Not fully implemented in UI, but backend ready:
- When user creates post, system suggests best board
- Based on content analysis
- Reduces manual organization effort

FEATURE 7: Trend Detection
---------------------------
Not fully implemented in UI, but backend ready:
- Fetches trending content in user's niche
- Provides post ideas
- Example: "Cottagecore kitchens trending this week"

7. TROUBLESHOOTING
================================================================================

ERROR: "Build failed - Cannot resolve symbol"
----------------------------------------------
Solution:
1. Click "Sync Now" when prompted
2. File → Invalidate Caches → Restart
3. Build → Clean Project
4. Build → Rebuild Project

ERROR: "Failed to connect to API"
---------------------------------
Causes & Solutions:
1. Backend not running:
   - Start your server (node server.js)
   - Or deploy to Heroku

2. Wrong API URL:
   - Check di_module.kt has correct baseUrl
   - For emulator: http://10.0.2.2:5000/
   - For real device: http://192.168.x.x:5000/

3. Network permission missing:
   - Check AndroidManifest.xml has:
     <uses-permission android:name="android.permission.INTERNET" />

4. CORS issue:
   - Ensure backend has CORS enabled:
     app.use(cors());

ERROR: "API key invalid"
------------------------
Solution:
1. Get OpenAI API key from https://platform.openai.com
2. Add to backend .env file
3. Restart server

ERROR: "App crashes on launch"
-------------------------------
Solutions:
1. Check Logcat for error messages:
   View → Tool Windows → Logcat

2. Common causes:
   - Missing permission declaration
   - Database migration issue
   - Missing dependency

3. Clean and rebuild:
   Build → Clean Project
   Build → Rebuild Project

ERROR: "Database corruption"
-----------------------------
Solution:
1. App → Device File Explorer → /data/data/com.pinterest.optimizer/
2. Delete "databases" folder
3. Reinstall app

App runs but UI looks broken:
----------------------------
Solutions:
1. Update compileSdk to 34 in build.gradle
2. Ensure Material 3 dependency is installed
3. Rebuild and clear app data
