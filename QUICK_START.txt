Benefits:

Drawbacks:

- Can't generate based on actual content

Quick Option B: Deploy Backend to Heroku (Takes 10 minutes)
───────────────────────────────────────────────────────────
1. Create Heroku Account
   └─ Go to: https://www.heroku.com/
      Sign up (free account OK)

2. Install Heroku CLI
   └─ Download from: https://devcenter.heroku.com/articles/heroku-cli

3. Create Backend Folder
   └─ Open terminal/command prompt:
      mkdir pinterest-backend
      cd pinterest-backend
      npm init -y

4. Install Dependencies
   └─ npm install express cors dotenv axios body-parser

5. Create server.js
   └─ Copy BACKEND_SETUP.js content
      Rename to server.js

6. Create .env file
   └─ PORT=5000
      OPENAI_API_KEY=sk-xxxxx

7. Get OpenAI Key
   ├─ Go to: https://platform.openai.com/api-keys
   ├─ Create new secret key
   ├─ Copy and paste into .env file
   └─ DO NOT share this key!

8. Test Backend Locally
   └─ node server.js
      Should print: "Server running on port 5000"

9. Deploy to Heroku
   └─ git init
      heroku login
      heroku create YOUR-APP-NAME
      git add .
      git commit -m "Initial commit"
      git push heroku main

10. Update Android App
    └─ In di_module.kt, change:
       .baseUrl("https://YOUR-APP-NAME.herokuapp.com/")

Done! Backend is live and app will use real APIs.

PART 5: TROUBLESHOOTING
================================================================================

Issue: "Cannot resolve symbol" in Android Studio
Solution:
1. File → Invalidate Caches → Restart
2. Click "Sync Now" when prompted
3. Build → Clean Project
4. Build → Rebuild Project

Issue: "Unresolved reference" in .kt files
Solution:
1. Check file is in correct folder:
   - models.kt → data/models/
   - dao.kt → data/database/
   - api.kt → data/api/
   Etc.

2. Check package declaration at top:
   package com.pinterest.optimizer.data.models

3. Gradle might be out of sync - try:
   File → Sync Now

Issue: App crashes on launch
Solution:
1. Check Logcat for error:
   View → Tool Windows → Logcat
2. Search for red text
3. Common causes:
   - Missing @AndroidEntryPoint annotation
   - Missing database migration
   - Incorrect manifest
4. If stuck, create fresh project and recopy files carefully

Issue: "Failed to connect" when generating captions
Solution:
1. Backend not running - start it:
   node server.js

2. Wrong API URL in di_module.kt:
   For emulator: http://10.0.2.2:5000/
   For real device: http://192.168.1.X:5000/ (your computer IP)
   For Heroku: https://app-name.herokuapp.com/

3. Network permission missing:
   Check AndroidManifest.xml has:
   <uses-permission android:name="android.permission.INTERNET" />

4. CORS disabled on backend:
   Add to server.js:
   app.use(cors());

Issue: Database errors
Solution:
1. Uninstall app: adb uninstall com.pinterest.optimizer
2. Clear app data: Settings → Apps → Pinterest Optimizer → Storage → Clear All
3. Reinstall app
