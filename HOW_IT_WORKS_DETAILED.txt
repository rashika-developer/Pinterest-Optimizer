1. CAPTION GENERATION FLOW
================================================================================

User Flow in App:
1. User enters: "Modern white kitchen with marble countertops"
2. Taps "Generate Captions"
3. Sees 3 caption options appear
4. Selects one favorite

What Happens Behind the Scenes:

STEP 1: User Input Collection
------------------------------
PostCreationScreen.kt collects text:
```kotlin
viewModel.generateCaptions(imageDescription) // Pass user input
```

STEP 2: ViewModel Processing
------------------------------
PostCreationViewModel.kt:
```kotlin
fun generateCaptions(description: String) {
    viewModelScope.launch {
        _captionState.value = CaptionState.Loading
        try {
            nicheRepository.getCurrentProfile().collect { profile ->
                if (profile != null) {
                    // Get current niche (e.g., "Home Decor")
                    val response = contentGenRepository.generateCaptions(
                        description,
                        profile.niche
                    )
                    _captionState.value = CaptionState.Success(response)
                }
            }
        } catch (e: Exception) {
            _captionState.value = CaptionState.Error(e.message ?: "Unknown error")
        }
    }
}
```

What's happening:
- Retrieves user's niche from database
- Makes API call with: description + niche
- Updates UI state to show loading spinner
- Updates UI with results when received

STEP 3: API Call (Android → Backend)
-------------------------------------
repository.kt makes HTTP request:

```
POST /api/generate-captions
Content-Type: application/json

{
    "description": "Modern white kitchen with marble countertops",
    "niche": "Home Decor",
    "tone": "engaging"
}
```

Behind the scenes (Retrofit):
- Converts request to HTTP POST
- Serializes data to JSON
- Sends to backend server
- Waits for response

STEP 4: Backend Processing
---------------------------
Backend server (server.js):

```javascript
app.post('/api/generate-captions', async (req, res) => {
    const { description, niche } = req.body;
    
    // Call OpenAI API
    const response = await axios.post(
        'https://api.openai.com/v1/chat/completions',
        {
            model: "gpt-3.5-turbo",
            messages: [
                {
                    role: "system",
                    content: `You are a Pinterest expert for ${niche}.
                    Generate 3 different caption styles: minimal, engaging, and sales-oriented.`
                },
                {
                    role: "user",
                    content: `Create captions for: ${description}
                    Return ONLY valid JSON with keys: minimal, engaging, sales`
                }
            ],
            temperature: 0.7
        },
        {
            headers: { 'Authorization': `Bearer ${OPENAI_API_KEY}` }
        }
    );
    
    // Extract AI response
    const content = response.data.choices[0].message.content;
    const parsed = JSON.parse(content);
    
    // Send back to Android
    res.json(parsed);
});
```

OPENAI API INTERACTION:
-----------------------
Backend sends prompt to OpenAI:
```
System: "You are a Pinterest expert for Home Decor. Generate 3 captions..."
User: "Create captions for: Modern white kitchen with marble countertops"
```

OpenAI responds with something like:
```
{
    "minimal": "Marble elegance. Where simplicity meets luxury.",
    "engaging": "Just finished our kitchen renovation! The marble countertops have completely transformed the space. What's your favorite part?",
    "sales": "This marble countertop transform is stunning! Shop similar products - link in comments 👇"
}
```

STEP 5: Response Back to Android
---------------------------------
Backend sends response:
```
HTTP 200 OK
Content-Type: application/json

{
    "minimal": "Marble elegance. Where simplicity meets luxury.",
    "engaging": "Just finished our kitchen renovation!...",
    "sales": "This marble countertop transform is stunning!..."
}
```

STEP 6: UI Update
-----------------
```kotlin
// In PostCreationViewModel

// PostCreationScreen recomposes and shows:
```

Backend:
```javascript
app.post('/api/generate-hashtags', async (req, res) => {
    const { content, niche } = req.body;
    
    // Step 1: Get trending hashtags from Pinterest
    const trendingHashtags = await fetchPinterestTrends(niche);
    
    // Step 2: Generate relevant hashtags with OpenAI
    const response = await openai.createChatCompletion({
        messages: [
            {
                role: "system",
                content: `Generate 15 relevant hashtags for ${niche}.
                Include these trending ones: ${trendingHashtags.join(', ')}`
            },
            {
                role: "user",
                content: `For this content: "${content}"`
            }
        ]
    });
    
    // Step 3: Parse and return
    const parsed = JSON.parse(response.choices[0].message.content);
    
    res.json({
        hashtags: parsed.hashtags, // 15 relevant hashtags
        trending: parsed.trending   // Top 3 trending hashtags
    });
});
```

How Trends Work:
1. Backend monitors Pinterest trends daily

3. Stores in database by niche
4. When generating hashtags, includes trending ones
5. User sees which hashtags are currently hot

4. DATABASE STORAGE FLOW
================================================================================

Local Data (Stored on Phone):

```
Room Database
├── niche_profiles (1 table)
│   ├── id (primary key)
│   ├── niche (selected by user)
│   ├── description
│   └── createdAt
│
├── pinterest_posts (1 table for each post user creates)
│   ├── id
│   ├── imageUri
│   ├── description (user input)
│   ├── captionMinimal (generated)
│   ├── captionEngaging (generated)
│   ├── captionSales (generated)
│   ├── hashtags (comma-separated)
│   ├── productLinks (JSON)
│   ├── boardName (suggested)
│   ├── scheduledTime
│   ├── isPosted (true when posted to Pinterest)
│   └── createdAt
│
├── analytics (tracked from Pinterest)
│   ├── id
│   ├── postId (reference)
│   ├── views
│   ├── saves
│   ├── clicks
│   ├── impressions
│   ├── engagementRate
│   └── updatedAt
│
└── trends (downloaded from backend)
    ├── id
    ├── title
    ├── description
    ├── category (niche)
    ├── trendScore
    └── discoveredAt
```

How Data Flows:

1. User creates post:
   ```
   PostCreationScreen (UI input)
   ↓
   PostCreationViewModel (state management)
   ↓
   ContentGenerationRepository (API calls)
   ↓
   Backend API (OpenAI, product finding)
   ↓
   Response back to Android
   ↓
   Room Database storage (PinterestPostDao)
   ↓
   Next time user opens app → Data loaded from database
   ```

2. Data persists locally:
   - Captions saved (can edit later)
   - Products saved (can reuse)
   - Posts saved (can reschedule)
   - Hashtags saved (can copy-paste)

3. Sync with cloud (optional):
   ```kotlin

   ```

STEP 6: Background Job Execution
```kotlin
class PostSchedulerWorker : Worker() {
    override fun doWork(): Result {
        // Every 15 minutes:
        val scheduledPosts = getScheduledPosts() // From DB
        
        scheduledPosts.forEach { post ->
            if (System.currentTimeMillis() >= post.scheduledTime) {
                // Time to post!
                postToPinterest(post)
            }
        }
        return Result.success()
    }
    
    private suspend fun postToPinterest(post: PinterestPost) {
        val response = api.createPinterestPin(
            imageUrl = post.imageUri,
            description = post.caption + "\n\n" + post.hashtags,
            boardId = post.boardName,
            link = post.productLinks[0].url // Link to first product
        )
        
        // Mark as posted
        post.isPosted = true
        updatePostInDatabase(post)
    }
}
```

7. ERROR HANDLING & RETRY LOGIC
================================================================================

What if API Call Fails?

```kotlin
try {
    val response = contentGenRepository.generateCaptions(description, niche)
} catch (e: IOException) {
    // Network error - no internet
    _captionState.value = CaptionState.Error("Check your connection")
    
    // Auto-retry in 5 seconds
    delay(5000)
    generateCaptions(description)
    
} catch (e: HttpException) {
    // Server error (500, 503, etc.)
    _captionState.value = CaptionState.Error("Server error. Try again later")
    
    // Don't retry immediately - wait 30 seconds
    delay(30000)
    
} catch (e: JsonException) {
    // API returned invalid JSON
    _captionState.value = CaptionState.Error("Invalid response format")
    // Don't retry - data is corrupted
}
```

Offline Mode:

```kotlin
// If no internet, show cached results
if (isOffline) {
    val cachedCaptions = database.getCachedCaptions(description)
    if (cachedCaptions != null) {
        _captionState.value = CaptionState.Success(cachedCaptions)
        showNotification("Using cached results - sync when online")
    } else {
        _captionState.value = CaptionState.Error("Offline - no cached data")
    }
}
```

SUMMARY OF DATA FLOW
================================================================================

Request → API → Processing → Response → Storage → Display → Action

Example: Generate Captions
──────────────────────────
1. User enters text (Android UI)
2. Android app sends HTTP POST (Retrofit)
3. Backend receives request (Node.js)
4. Backend calls OpenAI API (GPT-3.5)
5. OpenAI generates 3 captions (AI)
6. Backend returns JSON response (Backend)
7. Android receives response (HTTP)
8. Response saved to database (Room)
9. UI updated with captions (Compose)
10. User selects favorite (User)
11. Selected caption saved (Database)
12. User posts to Pinterest (Pinterest API)

Each step is crucial. Break one link and the flow stops.

This is why error handling and retry logic are important!
